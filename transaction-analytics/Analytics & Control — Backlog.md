# Analytics & Control — Backlog

Підтверджений перелік функціоналу для перетворення трекера на повноцінну систему
фінансового контролю: аналітика, бюджети, попередження, поради.
Плани реалізації — у папці `Plans/` (по одному на блок).

> [!warning] Передумова
> Спершу закриваються кроки 4–6 поточного roadmap (Crypto CSV → метчинг → estimate) —
> це ядро продукту. Backlog нижче йде після них (Bank CSV — крок 7 — піднято сюди як Plan 00).

## 0. Повнота даних
- **Bank CSV імпорт** (Privat + generic parser): Windows-1251, формати дат,
  кома-десятковий роздільник; детекція формату окремо від мапінгу. → [[Plan 00 — Bank CSV]]

## 1. Фундамент (без нього аналітика неможлива)
- **Категоризація**: ієрархічні категорії; автомапінг MCC → категорія (Monobank дає MCC);
  rules engine (description/merchant → категорія); ручний override, який завжди виграє
  і переживає ресинк. → [[Plan 01 — Категоризація]]
- **Нормалізація мерчантів**: «АТБ #123» і «ATB 456» → один контрагент. → [[Plan 01 — Категоризація]]
- **Детекція переказів між власними рахунками** — щоб переказ собі не рахувався як
  витрата+дохід. → [[Plan 02 — Гігієна даних]]
- **Курси НБУ**: сервіс курсів на дату + кеш; приведення до базової валюти (UAH).
  Той самий сервіс обслуговує крок 6 (estimate). → [[Plan 02 — Гігієна даних]]
- **Детекція регулярних платежів** (підписки): мерчант + сума + період. → [[Plan 02 — Гігієна даних]]

## 2. Аналітика
- Витрати/доходи по категоріях, місяцях, рахунках; MoM і порівняння з середнім за 3–6 міс.
- Cash flow: приплив/відтік, savings rate.
- Net worth у часі (баланси рахунків + крипта за поточним курсом).
- Крипто: реальна собівартість (з метчингу), згодом unrealized PnL (через `tradeRef`).
- Топ мерчантів, структура витрат, динаміка підписок.
→ [[Plan 03 — Агрегації]]

## 3. Контроль
- **Бюджети**: ліміт на категорію/місяць, факт з агрегацій, статус + темп (pace).
- **Цілі накопичення**: ціль + автотрекінг прогресу.
- **Календар регулярних платежів**: очікувані списання на N днів уперед.
→ [[Plan 04 — Бюджети й цілі]]

## 4. Попередження (rule-based)
- Перевищення 80%/100% бюджету; аномальна витрата (вище baseline по категорії);
  велика одинична транзакція; нова підписка; подвійне списання; сплеск комісій.
- Доставка: **Telegram-бот** (миттєві алерти + щотижневий/місячний дайджест).
→ [[Plan 05 — Alerts і Telegram]]

## 5. Поради (insights)
- Rule-based: «Ресторани +40% до середнього за 3 міс», «savings rate впав з 25% до 10%».
- LLM-шар: місячний огляд з **агрегатів** (не сирих транзакцій), вимикається конфігом.
→ [[Plan 06 — Insights]]

## Спільні правила (успадковані)
- Усі інваріанти vault діють: minor units, UTC, ідемпотентність, «нове — збоку».
- Definition of Done кожного плану = код + тести (unit + інтеграційний) + `tsc` чистий +
  жоден інваріант не порушено + результат можна побачити ізольовано.
- Секрети (Telegram token, LLM key) — лише env (NR3).
