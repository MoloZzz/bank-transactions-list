# Plan 03 — Шар агрегацій (analytics)

## Мета
Єдине джерело обчислених показників, з якого живляться звіти, бюджети, алерти й insights.

## Scope
- In: analytics-модуль з запитами-агрегаціями + експорт зведення (Sheets-аркуш або JSON CLI).
- Out: веб-дашборд; realtime.

## Показники
- Витрати/доходи по категорії × місяць × рахунок (у UAH через `toBaseUah`).
- MoM-зміна і відхилення від середнього за 3/6 міс по категорії.
- Cash flow за місяць: inflow, outflow, net; **savings rate** = net/inflow.
- Net worth: сума балансів рахунків + крипта за поточним курсом (знімок у часі).
- Крипто: собівартість з метчингу (залежить від roadmap кроку 5).
- Топ-N мерчантів за період; сума активних підписок/міс.

## Кроки
1. `AnalyticsModule` + `AnalyticsService` з методами-запитами (QueryBuilder/SQL).
   Фільтри: `isInternalTransfer = false`, тип, період (групування за **локальним** часом,
   зберігання UTC — [[Invariants]] #2).
2. DTO-и зведень; CLI `npm run report -- --month=2026-06` → JSON/консоль.
3. Експорт місячного зведення в окремий аркуш Google Sheets (існуючий Sheets-клієнт).
4. Снапшот net worth: job пише `NetWorthSnapshot(date, totalUahMinor, breakdown jsonb)`.

## Критерії приймання
- [ ] Числа збігаються з ручним SQL на контрольній вибірці (задокументувати запит у тесті).
- [ ] Внутрішні перекази виключені; доходи/витрати не змішані.
- [ ] Місячні межі коректні для локального часу (тест: транзакція 23:30 UTC останнього дня).
- [ ] Все в цілих мінорних одиницях; ділення (savings rate) — контрольована точність.
- [ ] Unit на кожен метод агрегації (фікстури), інтеграційний проти Postgres.
- [ ] `tsc` чистий, наявні тести зелені.
